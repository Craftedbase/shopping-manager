# 買い物管理アプリ

購入ログ、在庫、最安値店舗、買うものチェックリストをスマホ優先で管理する静的Webアプリです。

## 主な機能

- 商品・店舗・単位設定
- 商品画像の登録
- 税込価格の購入ログ登録
- 商品単位をもとにした単位価格の自動計算
- 商品ごとの最安値店舗分析
- 在庫の手動管理
- 在庫不足からの買うものチェックリスト自動表示
- 手動の買うもの追加・チェック済み管理
- 店舗別購入履歴一覧
- 商品別購入店舗履歴一覧
- Google KeepやLINEに貼り付けやすい共有テキスト生成
- PWA対応

## データ保存について

このアプリのデータは、利用しているスマホやブラウザの `localStorage` に保存されます。

- GitHubには買い物データは保存されません
- 端末ごと、ブラウザごと、URLごとにデータは別です
- ブラウザのサイトデータを削除すると、アプリ内データも削除されます
- 家族共有は、共有テキストをコピーまたは共有する方式です

## GitHub Pagesで公開する手順

1. GitHubで新しいリポジトリを作成します。
2. このフォルダ内のファイル一式をリポジトリにアップロードします。
3. GitHubのリポジトリ画面で `Settings` を開きます。
4. 左メニューの `Pages` を開きます。
5. `Build and deployment` の `Source` で `Deploy from a branch` を選びます。
6. `Branch` で `main`、フォルダは `/root` を選び、保存します。
7. 数分後に表示されるGitHub PagesのURLをスマホで開きます。
8. スマホのブラウザメニューから「ホーム画面に追加」します。

## 公開に必要なファイル

最低限、以下のファイルとフォルダを公開してください。

- `index.html`
- `styles.css`
- `app.js`
- `manifest.json`
- `service-worker.js`
- `icons/`
- `.nojekyll`

## PWAの注意点

PWAのService Workerは `file://` では動作しません。GitHub PagesなどのHTTPS URLで開いた場合に有効になります。

アプリを更新したあと、スマホ側に古いキャッシュが残る場合があります。その場合はブラウザでページを再読み込みするか、サイトデータを削除してから開き直してください。

## ローカルで確認する場合

`index.html` をブラウザで開くと基本機能は確認できます。ただし、PWAのインストールやオフラインキャッシュはHTTPSまたはlocalhost環境で確認してください。
